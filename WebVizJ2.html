Introdução¶

Neste projeto realizada a modelagem, execuo e posterior anlise do problema proposto em https://ieeexplore.ieee.org/abstract/document/6629989 .

Para tanto, so seguidos os seguintes passos:

  1. Primeiramente so gerados os dados, que diferentemente do descrito no artigo, so gerados artificialmente:
    1. $n$ grupos de retngulos, para $n \in [3,7]$ ($n$ retirado de uma distrubuio uniforme em $[3,7]$);
    2. cada grupo contando com $k$ retngulos, com $k \in [5,15]$ ($k$ retirado de uma distrubuio uniforme em $[5,15]$)
    3. cada retngulo tem uma altura e comprimento em $[2,4]$ (tambm, retirado de uma distribuio uniforme)
  2. Depois de uma breve discusso quanto ao modelo usado, os retngulos tm suas posies alteradas, de forma a minimizar sobreposio, enquanto mantm a estrutura de vizinhana;
  3. Diversos mtodos de otimizao so utilizados, e seus resultados em termos de resultado obtido e tempo de execuo so ento comparados.

Depois de uma primeira execuo, com breve discusso, os passos so repetidos diversas vezes, para se obter execues com valores diferentes, afim de obtermos uma ideia melhor dos desempenhos dos mtodos de otimizao usados.

Os trechos de cdigos utilizados para tal execuo so apresentados, com uma breve explicao.

Modelagem Matemática:¶

Para atingirmos os objetivos delineados na introduo, buscamos minimizar a "funo energia":

$$ E = (1-\alpha)E_o + \alpha E_N,\\ $$

para $\alpha \in (0,1)$, onde $E_o$ referente taxa de sobreposio dos retngulos e $E_N$ a quanto a soluo obtida distorce a estrutura de vizinhana original dos dados. Mais precisamente, temos

$$ E_o = \frac{2}{n(n+1)} \sum_{i=1}^n\sum_{j=1}^n [O_{ij}(x,h)O_{ij}(y,v)]\\ $$

onde

$$ O_{ij}(x,h) = \left\{ \begin{array}\\ \frac{1}{h_j^4} [h_j^2 - (x_i - x_j)^2]_+,\quad \textrm{se} \quad & x_i \ge x_j\\ \frac{1}{h_i^4} [h_i^2 - (x_i - x_j)^2]_+,\quad \textrm{se} \quad & x_i < x_j \end{array} \right. $$

e

$$ E_N = \frac{n^2}{2(\|\delta_x\|^2 + \|\delta_y\|^2)} \left(\|Lx - \omega\delta_x\|^2 + \|Ly - \omega\delta_y\|^2\right) $$

onde

$$ \delta_x = Lx^0, \quad \textrm{e} \quad \delta_y = Ly^0 $$

sendo

$$ L_{ij} = \left\{ \begin{array}\\ \frac{-1}{|i|}, &\mbox{se } j \neq i \\ 1, & \mbox{se } i = j\\ 0, & \mbox{caso contrrio.} \end{array} \right. $$

As implementaes dessas funes so dadas a seguir.

x(x) = x  0 ? x : 0

O(i, j, x, h) = x[i]  x[j] ? x( h[j]^2 - (x[i] - x[j])^2 )^2 / h[j]^4 : 
                                x( h[i]^2 - (x[i] - x[j])^2 )^2 / h[i]^4
 
function E(xy, h, w)
    n = size(xy)[2]
    
    x = xy[1:2:end]
    y = xy[2:2:end]  
    retval = 0
    
    for i in 1:n     
        for j in (i + 1):n
            retval +=  O(i, j, x, h) *  O(i, j, y, w)
        end
    end 
    return 2 * retval / (n * (n + 1))
end

Breve descrição e comparação dos resultados dos métodos¶

A seguir realizada a execuo e posterior comparao de diversos mtodos de otimizao para o problema discutido. Em particular, so executados:

  1. LBFGS ("limited memory BFGS") com busca linear de Hager Zhang (como exposto no artigo $\textit{W. W. Hager and H. Zhang (2006) Algorithm 851: CG_DESCENT, a conjugate gradient method with guaranteed descent}$)
  2. LBFGS com busca linear de Armijo (StrongWolfe, com o parmetro c_1 = 1e-4)
  3. "Descida por gradiente" com busca linear de Hager Zhang
  4. "Descida por gradiente" com busca linear de Armijo, como discutido em aula
  5. "Descida por gradiente com momento" e
  6. "Simmulated annealing"

No mtodo BFGS (BroydenFletcherGoldfarbShanno) um mtodo iterativo para a resoluo de problemas de otimizao, preferencialmente sobre funes duplamente diferenciveis (apesar de tambm poder apresentar performance boa mesmo para problemas no suaves). um mtodo do tipo quase-newton, incorrendo no clculo de aproximao da Hessiana da funo objetivo, sendo, entre esses, um dos mtodos mais populares. Neste mtodo, a direo de busca calculada como $B_k p_k = - \nabla f(x_k)$, onde $B_k$ uma aproximao da Hessiana. Uma busca linear realizada na direo $p_k$ ento usada para encontrar $x_{k+1}$, minimizando $f(x_k + \gamma p_k)$, com $\gamma > 0$. A $B_{k+1}$ aproximada como:

$$ B_{k+1} = B_k + \frac{y_k y_k^T}{(y_k,s_k)} + \frac{B_k s_k s_k^T B_k^T}{s_k^T B_k s_k} $$

onde $y_k = \nabla f(x_{k+1}) - \nabla f(x_k)$ e $s_k = x_{k+1} - x_k$.

O mtodo LBFGS (algoritmo Limited BroydenFletcherGoldfarbShanno) similar ao BFGS, exceto que, enquanto no BFGS calculada uma aproximao Hessiana como uma matriz densa, no LBFGS apenas alguns vetores so armazenados, e a aproximao representada implicitamente, tornando-o adequado para problemas com muitas variveis.

No mtodo de "Descida por gradiente com momento", adicionado um $\gamma \in (0,1)$ em $x_{n+1} =\gamma x_n \eta \nabla f(xn)$.

O mtodo de Nelder Smead aqui usado baseado na implementao de Nelder and Mead (1965) e Gao and Han (2010). Ele o mtodo padro que Optim utiliza para minimizao quando o usurio no indica que outro deve ser usado. um mtodo que no faz uso do gradiente da funo objetico (sendo, assim, especialmente adequado para quando tal funo no diferencivel) e um mtodo de busca direta. Ele mantm um registro de um simplexo (isto , uma quantidade de pontos que gera um simplexo no espao do problema) e, a cada iterao, o atualiza segundo uma de quatro operaes reflexo, expanso, contrao ou encolhimento, o objetivo sendo iterativamente trocar o pior ponto do simplexo por um ponto melhor (o valor minimizante retornado ou um dos pontos, ou o centroide do simplexo).

$\textit{Simmulated annealing}$ um algortmo probabilistico, baseado no algortmo de Metropolis-Hastings, para encontrar o mnimo de uma funo, mais frequentemente sobre domnios grandes. Ele foi adicionado aqui mais como um "problema teste", sendo um mtodo no adequado ao problema considerado. Nota-se que, para esse mtodo, foi utilizado um limite de iteraes 10 vezes maior do que para os demais mtodos, porque cada iterao dele muito rpida, e manter o mesmo limite de iteraes dificultaria a anlise do tempo levado para os demais mtodos - mas os valores mnimos obtidos por esse mtodo foram excepcionalmente ruins, como visto adiante.

So executadas 6 iteraes, a fim de fazer uma anlise do desempenho de cada mtodo para diferentes casos. Assim, a cada iterao so gerados dados diferentes e cada um dos algoritmos executado nesses dados. Todas as execues so com $\alpha = 0,5$ e $\omega = 2$.

Considerações gerais¶

Nota-se que os mtodos LBFGS no geral tendem a tempo e memria consideravelmente maiores do que o BFGS nas instncias consideradas do problema. Isso indica que o problema considerado no atinge um tamanho grande suficiente para obter vantagem da limitao de memria do LBFGS.

Nota-se tambm que os mtodos utilizados sem diferenciao automtica tomaram consideravelmente mais tempo de processamento, mas apreciavelmente menos memria.

Nota-se, ademais, que h uma grande variao para os problemas: alguns so muito "difceis", e outros muito "fceis" no geral.
